# 🚀 Zero to Operational: Setup Guide for ThalAI Guardian

This guide provides step-by-step instructions to get the **ThalAI Guardian v8.0** platform running on your local machine from scratch.

---

## 📋 Prerequisites

Ensure you have the following installed:

- **Node.js** (v18.0 or higher) & **npm**
- **Python** (v3.8 or higher)
- **MongoDB** (Local instance or [Atlas URI](https://www.mongodb.com/cloud/atlas/lp/try2))
- **Git**

---

## 🛠️ Step 1: Clone and Infrastructure

```bash
# Clone the repository
git clone <repository-url>
cd thalai-guardianV8

# Install dependencies for all modules (Backend, Frontend, AI Service) in one command
npm install
npm run install-all
```

---

## 🔐 Step 2: Environment Configuration

Create a `.env` file in the **root directory**. This file will be the single source of truth and can be synced to all modules automatically.

```env
# --- SERVER CONFIG ---
PORT=5000
NODE_ENV=development
JWT_SECRET=THALAI_SECRET_KEY_2026_CHANGE_ME
FRONTEND_URL=http://localhost:3000

# --- DATABASE ---
MONGODB_URI=mongodb://localhost:27017/thalai-guardian

# --- AI HUB ---
AI_SERVICE_URL=http://localhost:8000

# --- NOTIFICATION SYSTEM (CRITICAL) ---
# For Email (Gmail App Password)
EMAIL_SERVICE=gmail
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-16-character-app-password

# For SMS (Twilio)
TWILIO_ACCOUNT_SID=your_sid
TWILIO_AUTH_TOKEN=your_token
TWILIO_PHONE_NUMBER=your_number

# --- EXTERNAL INTEGRATIONS ---
ERAKTKOSH_API_KEY=your_key
GEMINI_API_KEY=your_gemini_key
```

**Sync the variables to all sub-folders:**

```bash
npm run setup-env
```

## 🚧 Common Fixes & Troubleshooting

### 1. Port 5000 is already in use

If you get an `EADDRINUSE` error on port 5000 (common on MacOS/Windows with AirPlay or VSCode):

- **Windows**: `npm run kill-port` (if available) or `netstat -ano | findstr :5000` then `taskkill /PID <PID> /F`
- **Alternative**: Change `PORT=5001` in your `.env` and re-run `npm run setup-env`.

### 2. Frontend Blank Screen (White Screen)

If the frontend stays blank in a production build, ensure you have removed `manualChunks` from `vite.config.js`. I have fixed this in v8.1.0 to ensure stable bundles.

### 3. AI Service Fails to Start

Ensure you have installed the requirements:

```bash
cd thalai-ai-service
pip install -r requirements.txt
python app.py
```
